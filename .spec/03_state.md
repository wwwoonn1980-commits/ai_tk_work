# 에이전트 상태 명세서 (State Specification)

## 1. 개요
State 컴포넌트(`AgentState` 등)는 에이전트의 모든 데이터와 상태를 담고 있는 단일 진실 공급원(Single Source of Truth)입니다. 도구 간의 데이터 전달은 이 State 객체를 매개로 이루어져야 합니다.

## 2. 상태 데이터 구조 (Properties)
구현체는 최소한 다음의 정보들을 보관할 수 있는 구조여야 합니다.

- **작업 경로 정보**: 입력 PDF 경로, 엑셀 템플릿 경로, 최종 출력 경로
- **실행 옵션**:
  - `includeAll` (boolean): 파싱 도구가 요구사항을 수집할 범위를 결정하는 플래그.
    - `false` (기본값): 정형 패턴(예: 'Ⅳ.3 요구사항 상세' 등 사전 정의된 섹션)에 해당하는 요구사항만 수집한다.
    - `true`: 정형 패턴 외에 비정형/서술형 요구사항 후보까지 모두 수집한다.
- **작업 메타데이터**: 전체 페이지 수(`totalPages`), 현재 진행 페이지(`currentPage`) 등.
  - `totalPages`는 `pdfTool`이 PDF를 연 직후 1회 갱신한다.
  - `currentPage`는 `parseTool`이 페이지 루프를 돌면서 매 페이지 진입 시 갱신한다.
- **작업 상태**: 현재 상태 플래그 (`idle`, `running`, `completed`, `error`)
- **로그 데이터**: 에이전트가 남긴 작업 이력 문자열의 배열 (`logs`)
- **요구사항 저장소**: 추출된 요구사항 객체들을 저장하는 공간. 고유 ID를 기반으로 중복 검사 및 병합이 가능하도록 `Map` 또는 `Dictionary` 자료구조 사용을 권장합니다.

## 3. 필수 인터페이스 (Methods)

### 3.1. 로깅 (예: `addLog(message)`)
- 시스템 시간과 메시지를 결합하여 내부 로그 배열에 저장하고, 실시간 모니터링을 위해 콘솔 등에 출력하는 기능을 제공해야 합니다.

### 3.2. 요구사항 추가 및 병합 (예: `addRequirement(req)`)
- 요구사항 객체를 입력받아 저장소에 추가합니다.
- **병합(Merge) 로직 필수**: 
  - 동일한 ID를 가진 요구사항이 이미 저장소에 존재한다면 덮어쓰지 않고 필드를 병합해야 합니다.
  - 이름, 타입, 카테고리 등은 비어있을 때만 채워 넣고, '상세 설명'이나 '세부 내용' 등 본문 텍스트는 기존 내용과 중복되지 않는 선에서 이어붙여야(concat) 합니다.

### 3.3. 결과 추출 (예: `getRequirementsList()`)
- 저장소(Map)에 축적된 모든 요구사항 데이터를 배열(Array) 형태로 변환하여 반환해야 합니다.
