# 에이전트 아키텍처 명세서 (Overview Specification)

## 1. 아키텍처 목적
본 프로젝트는 PDF에서 요구사항을 추출하여 엑셀(XLSX)로 변환하는 작업을 자율적으로 수행하는 에이전트 시스템을 구축하기 위한 명세입니다. 특정 코드 구현에 얽매이지 않고, 의사결정(Manager), 기억(State), 실행(Tools)의 관심사를 분리하여 향후 LLM이나 새로운 도구를 쉽게 교체/확장할 수 있는 유연한 구조를 지향합니다.

## 2. 핵심 컴포넌트 구성
에이전트 시스템은 다음 3가지 주요 컴포넌트로 구성되어야 합니다.

1. **Manager (Orchestrator)**: 전체 작업의 흐름을 제어하는 주체. 도구를 언제, 어떤 순서로 호출할지 결정하며 예외 상황에 대한 통제권을 갖습니다.
2. **State (Memory)**: 에이전트의 단기/장기 기억 장치. 작업 진행 상태, 추출된 데이터, 발생한 로그 등 모든 전역 데이터를 중앙 집중식으로 관리합니다.
3. **Tools (Capabilities)**: 특정 목적의 작업을 수행하는 개별 모듈. 상태(State)를 읽거나 수정할 수 있으며, 독립적으로 테스트 및 교체가 가능해야 합니다.

## 3. 에이전트 표준 라이프사이클
에이전트의 실행 흐름은 다음 단계를 반드시 준수해야 합니다.

1. **초기화 (Initialization)**: Manager가 생성될 때 초기 환경 설정(입출력 경로 등)을 바탕으로 빈 State 객체를 생성합니다.
2. **작업 시작 (Execution Start)**: State의 상태를 `running`으로 변경하고 작업을 시작합니다.
3. **도구 연쇄 호출 (Tool Chain Invocation)**:
    - 텍스트 추출 도구 호출 -> 결과 파싱 도구 호출 -> 세부내용 정제 도구 호출 -> 엑셀 저장 도구 호출
    - 각 도구는 완료 시 State를 업데이트하고 제어권을 Manager에게 반환합니다.
4. **결과 처리 및 종료 (Completion/Error)**:
    - 모든 도구가 정상 완료되면 State를 `completed`로 변경하고 최종 결과를 반환합니다.
    - 도중 오류가 발생하면 State를 `error`로 변경하고, 로깅 후 작업을 안전하게 중단합니다.
